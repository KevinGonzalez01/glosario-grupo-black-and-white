## Terminos Generales de Programacion
*(Algoritmo: Secuencia ordenada de pasos lógicos para resolver un problema.
Variable: Contenedor nombrado que almacena un valor de datos en memoria que puede cambiar.
Función: Bloque de código reutilizable que realiza una tarea específica.
Sintaxis: Conjunto de reglas que definen cómo se escribe y estructura el código en un lenguaje específico.
Código Fuente: Conjunto de instrucciones escritas por un programador en un lenguaje de alto nivel.
Compilador: Programa que traduce el código fuente a lenguaje de máquina (binario).
Bug: Un error, fallo o defecto en el software que produce un resultado inesperado.
Debugging (Depuración): Proceso de encontrar y corregir errores (bugs) en el código.
Estructuras de Control: Instrucciones que determinan el flujo de ejecución del programa, como condicionales (if/else) y ciclos (for/while).
Tipos de Datos: Clasificación de la información, siendo los más comunes:
Integer (Int): Números enteros.
Float: Números decimales.
String: Cadenas de texto.
Boolean (Bool): Valores lógicos true (verdadero) o false (falso).
Estructura de Datos: Forma organizada de almacenar y gestionar datos, como arreglos (arrays/matrices).
Programación Orientada a Objetos (POO): Paradigma basado en "objetos" que agrupan datos y comportamientos.
Clase: Plantilla para crear objetos, definiendo sus propiedades y métodos.
Framework: Conjunto estandarizado de conceptos, prácticas y herramientas enfocadas a un tipo de problema específico para facilitar el desarrollo.
Control de Versiones: Sistema (como Git) que registra cambios en el código a lo largo del tiempo.
API (Application Programming Interface): Conjunto de reglas que permite que una aplicación se comunique con otra.)*
